# Topic Dimensions

# Format Examples

以下は **形式のバリエーション** を示す例です。内容をそのまま使わず、形式の多様さを参考にしてください。

<example format="シチュエーション → 行動">
コードレビューで「LGTM」が一度も出なかったエンジニアが、翌日取った驚きの行動とは？
</example>

<example format="穴埋め">
git commit -m "〇〇" ← 深夜3時のコミットメッセージ
</example>

<example format="こんな〇〇は嫌だ">
こんなプルリクエストは絶対マージしたくない。どんなの？
</example>

<example format="もしも">
もし本番環境のDBに「DROP TABLE」を実行するボタンがオフィスの壁にあったら、何が起きる？
</example>

<example format="ランキング">
「エンジニアが一斉退職するきっかけ」ランキング、惜しくも2位だったのは？
</example>

<example format="架空の〇〇">
売れなさすぎて即打ち切りになった技術書のタイトルは？
</example>

<example format="対比・選択">
「ドキュメントのないコードベース」と「コードのないドキュメント」、配属されるならどっち？ 先輩はなんと言った？
</example>

# Anti-patterns（生成してはいけないもの）

- 「〇〇とは？」で終わるだけの単調な形式の連発
- 「衝撃の一言とは？」のテンプレの使い回し
- 上の例文そのものの焼き直し・微修正
- プログラミングを知らない人が完全に置いてけぼりになる高度すぎる専門ネタ
